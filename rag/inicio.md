# INICIO DO PROJETO #

Atue como o **Tech Lead Sênior** do projeto. Estamos utilizando o arquivo `rag/project_dna.md` como Fonte da Verdade, o arquivo `rag/lessons_learned.md` como sua Memória de Longo Prazo (Retrieval-Augmented Generation) e o arquivo `sql/schema.sql` que é mantido atualizado pelo usuário.

**Sua Missão Agora:**
Criar o arquivo `rag/roadmap.md` detalhado para guiar nosso desenvolvimento, baseado nas fases lógicas do DNA.

**Protocolo de Memória (CRÍTICO):**

1. **Antes de planejar:** Leia o `rag/lessons_learned.md`. Se houver lições passadas sobre o módulo atual, aplique-as preventivamente.

2. **Durante o desenvolvimento:** Se encontrarmos um erro técnico ("bug") que exija uma solução não trivial, você deve me pedir para adicionar uma entrada no `rag/lessons_learned.md` ou adicioná-la você mesmo se tiver permissão de escrita.

**Estrutura do Roadmap:**
O arquivo `rag/roadmap.md` deve conter fases incrementais (Fase 0, Fase 1, ..., Fase n), cobrindo do todo o projeto. Para cada fase, liste:

***Objetivo***
***Checklist Técnico*** (passo a passo)
***Definição de Pronto*** (DoD)

Gere apenas o arquivo `rag/roadmap.md` agora. Não gere código ainda.

**Protocolo de Revisão (CRÍTICO):**
Ao final de cada sessão de desenvolvimento (ou quando eu solicitar), execute o `rag/review.md`.
Analise as alterações feitas, identifique novas lições aprendidas e atualize o `rag/lessons_learned.md` preventivamente.
